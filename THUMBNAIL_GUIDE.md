## 一、設計核心原則

> **一眼3秒法則**：觀眾在推薦列表滑過時只有 0.5–1 秒決定點擊。
> 縮圖必須在手機螢幕（≈ 150×84px 縮略尺寸）傳達一個清晰訊號。

| 原則 | 說明 |
|---|---|
| 單一視覺焦點 | 一張縮圖只說一件事：人物情緒 OR 關鍵字，二選一主導 |
| 情緒優先數字 | 有人臉情緒的縮圖 CTR 通常高於純文字30–40%，Linara 半身必須出現 |
| 品牌一致性 | 色彩系統、字型、構圖邏輯全系列統一，讓觀眾一眼認出是 Luna's |
| 標題互補不重複 | 縮圖文字與標題不要完全相同，縮圖給情緒/問題，標題給答案/方向 |
| 行動測試 | 每支影片應準備2個縮圖版本（A/B），上線48小時後依CTR保留勝出版 |

## 二、技術規格

| 項目 | 規格 |
|---|---|
| 尺寸 | 1280 × 720 px（16:9）|
| 解析度 | 72 dpi（螢幕用）|
| 檔案格式 | JPG（主要）或 PNG（含透明背景元素時）|
| 檔案大小 | ≤ 2MB（YouTube 上傳限制）|
| 安全邊距 | 四邊保留 40px 不放任何重要元素（避免被 UI 裁切）|
| 字型最小尺寸 | 縮圖預覽狀態（150px寬）下仍需清晰可讀，建議正文字型 ≥ 80pt |

## 三、品牌色彩系統（唯一標準，禁止更改）

| 色彩名 | Hex | RGB | 用途 |
|---|---|---|---|
| 深藍 | `#0D0D2B` | 13, 13, 43 | 主底色（最常用）|
| 深紫 | `#1A1A3A` | 26, 26, 58 | 輔底色、漸層過渡、服裝主色 |
| 玫瑰金 | `#B4918F` | 180, 145, 143 | 標題字、邊框、命盤符號、高亮線條 |
| 霧白 | `#F0EEE8` | 240, 238, 232 | 副標字色、肌膚色、輔助文字 |
| 玉石青 | `#7BBFB5` | 123, 191, 181 | 奇門九宮高亮色（輔助用，不超過畫面10%）|

### 禁止色彩
- ⛔ 純黑 `#000000`（太死板，與品牌神祕感不符）
- ⛔ 純白 `#FFFFFF`（對比過強，失去質感）
- ⛔ 高飽和紅/黃/綠（帶貨主播感，破壞知性定位）
- ⛔ 超過3種顏色同時使用（視覺噪音）

**文字配置：**

**範例文字：**

## 五、Linara 人物使用規則

| 規則 | 說明 |
|---|---|
| 版型 | 虛擬人物半身（真人比例，風格化分身）；非動漫誇張造型 |
| 方向 | 面朝右：版型A/C（視線引導向文字）；面朝正面：版型B |
| 表情 | 冷靜帶洞察感為主；特殊場景（感情主題）可帶輕微溫柔感 |
| 服裝 | 深藏青色知性套裝 × 月白色寬領內搭（固定，不換）|
| 配件特徵 | 雙耳細圓圈耳環、右眼下方美人痣（保留真人特徵）|
| 道具 | 視內容選擇：命盤片（命理主題）/ 無道具（感情/心理主題）|
| 背景融合 | 人物邊緣做輕微發光/光暈效果（玫瑰金色調），增加神祕感 |
| 禁止 | 過度美化、卡通化、換裝、加文字在人物臉上 |

## 六、文字設計規則

### 字型選擇
| 用途 | 推薦字型 | 備選 |
|---|---|---|
| 中文主標題 | 思源黑體 ExtraBold / Noto Sans TC Black | 方正超粗黑 |
| 中文副標 | 思源黑體 Bold / Noto Sans TC Bold | 微軟正黑體 Bold |
| 英文品牌字 | Montserrat SemiBold / Raleway Bold | Futura |
| 命盤符號 | 自訂圖形（勿用文字替代）| — |

### 文字排版規則
- **主標字級**：在縮圖預覽（手機縮圖）下清晰可見，建議實際字級 ≥ 100pt
- **行距**：1.1–1.2倍（緊湊有力，避免鬆散）
- **字間距**：標題字 +50–100（拉開有氣場）
- **禁止**：超過3行文字、超過2種字型、文字壓在人物臉部
- **陰影**：主標題加輕微深色陰影（增加對比），不超過 2px blur

### 文字數量硬性限制
| 元素 | 最多字數 |
|---|---|
| 主標題（大字） | 5個中文字 |
| 副標/補充 | 8個中文字 |
| 品牌標記 | Luna's Astral Code（固定） |
| 總文字數 | ≤ 15個中文字（含品牌標記）|

## 七、A/B測試流程

```
準備階段（發佈前）：
  □ 製作版本A（主方案）
  □ 製作版本B（差異：換文字 或 換人物表情 或 換構圖）
  □ 兩版本除測試變量外其餘完全相同

測試階段（發佈後48小時）：
  □ 上線時用版本A
  □ 24小時後檢查CTR
  □ CTR < 2%：立即換成版本B
  □ CTR ≥ 2%：繼續觀察至48小時再判斷

評估標準：
  □ 新頻道前3個月：CTR ≥ 3% = 通過
  □ 第4個月起：CTR ≥ 4% = 通過，≥ 6% = 優秀
  □ 保留勝出版本，記錄原因供未來參考
```

## 八、常見錯誤清單

| ❌ 錯誤 | ✅ 正確做法 |
|---|---|
| 文字超過5個字（主標） | 砍到最核心痛點詞，≤5字 |
| 使用純黑/純白背景 | 用深藍 `#0D0D2B` |
| 人物太小看不清表情 | Linara 人物至少佔畫面40%以上 |
| 多種顏色雜用 | 只用品牌三色（深藍/深紫/玫瑰金） |
| 縮圖文字與標題完全相同 | 縮圖給問題/情緒，標題給方向/答案 |
| 無人出鏡純文字縮圖 | 必須有 Linara 人物 |
| 每支縮圖風格都不同 | 固定用三種版型之一 |
| 文字壓在人物臉部 | 文字與人物分區排布 |
