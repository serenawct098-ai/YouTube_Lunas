## 五、每支影片發佈 SOP（標準作業流程）

### 發佈前 3 天
```
□ 腳本完成（依 LINARA_IP_Design.md 口吻規則）
□ 命盤圖表 / 九宮格視覺準備完成
□ 縮圖初稿完成（Linara 半身 + 痛點詞 ≤ 5 字）
□ 標題公式套用：[核心命理詞]：[痛點詞]｜Luna's Astral Code
□ 描述初稿完成（前 150 字含長尾詞 + 承諾結果）
```

### 發佈當天
```
□ 上傳影片（1080p 以上）
□ 標題確認（品牌詞在標題末）
□ 描述貼入（首行關鍵詞 + 品牌簽名）
□ Tag 設定（3–5 個精準詞，無泛用大詞）
□ 加入對應 Playlist（24 小時內必完成）
□ 縮圖上傳（真人 + 命盤圖 + ≤ 5 字痛點詞）
□ 章節時間軸設定（Chapters）
□ 字幕上傳或開啟自動字幕後手動校正
```

### 發佈後 24 小時
```
□ IG @lunasastralcode 發佈對應 Reels/Stories，Caption 結尾附 YouTube Playlist 連結
□ 回覆前 20 則留言（觸發演算法互動訊號）
□ 記錄：首 24 小時 CTR / 完播率 / 展示次數（截圖存檔）
```

### 發佈後 7 天
```
□ 檢查數據：CTR < 4% → 換縮圖測試
□ 完播率 < 40% → 找 Drop-off 點，標記腳本問題
□ 把表現最好的關鍵字補入下一支影片描述
```

## 六、月度老片復活排程

> 每月第一週執行，不佔新片發佈名額

| 步驟 | 動作 | DoD |
|---|---|---|
| 1 | 選 3–5 支舊片健檢 | 確認標題含現時主推長尾詞 |
| 2 | 重做縮圖（換痛點詞） | 新縮圖 A/B 測試 7 天 |
| 3 | 更新描述前 150 字 | 補入當月主推關鍵字 |
| 4 | 加入最新 Playlist | 確認 Playlist 描述已更新 |
| 5 | 觀察 7 日展示次數 | +20% 以上 = 成功復活 |
