## 一、演算法核心（2026版）

### 核心轉變
| 舊邏輯 | 2026 新邏輯 |
|---|---|
| 訂閱數決定推播 | 觀眾滿意度決定推播 |
| 追蹤用戶社交圖譜 | 依興趣微群集（micro-niche cluster）分發 |
| Watch Time 為王 | Good Abandonment 合法（看完即滿足＝正訊號） |
| 標題 / 標籤決定排名 | Gemini AI 語意 Semantic ID 全文理解（口播＋字幕＋描述優先） |
| 長片佔優 | 6–10 分鐘高留存 > 20 分鐘低留存 |
| 單片爆發 | Repeated Exposure／Session Depth：同系列連看比單片更強 |
| Tags 決定分類 | Tags 已降至 minimal role，Title ＋ Description 前兩行為核心語意訊號 |

### 核心 KPI
- **CTR ≥ 6%**（縮圖＋標題組合）——新頻道前 3 個月容忍門檻 ≥ 3%，第 4 個月起恢復 6% 標準
- **平均觀看時長 ≥ 50%**
- **Search Satisfaction**：標題承諾必須在影片前 60 秒兌現；口播關鍵字須在前 60 秒自然出現
- **Session Depth ≥ 2**：觀眾看完後繼續在頻道看下一支（依賴 Playlist）
- **Co-Visitation Neighborhood**：觀眾最近 31 天觀看記錄決定演算法測試池；保持語意一致性可借力同微群集頻道的觸及

### AI 內容規則（強制）
- 使用 AI 語音（如 ElevenLabs）必須勾選「已更改的內容」揭露
- 真人出鏡命理內容具 Originality Score 優勢，優先採用
- SynthID / C2PA 違規可導致頻道降權或停利資格

## 三、格式 / 風格 / 視覺規範

### 影片結構 SOP（每支標準流程）

> **腳本方法論：故事型腳本 + 邏輯框架骨架**  
> - Hook = 情緒/故事共鳴先行  
> - 結構講解 = 邏輯框架骨架（概念拆解＋行為科學比例）  
> - 案例段 = 故事回到行為科學（讓觀眾感受＋理解）  
> - 收尾 = 溫暖重生，給行動方向  
> ⚑ 完整腳本範例請參見 LINARA_IP_Design.md §五

```
0–30 秒：Hook（故事型情緒共鳴）
  → 情緒問句＋直述痛點＋核心關鍵字在口播自然出現（≤ 30 秒內）
  → 例：「在生活上有部分人，明明努力了很久，但就是感覺，有一堵牆阻礙自己一樣。」
  → 禁止：Logo 動畫 / 片頭音樂 / 自我介紹

30–240 秒：結構講解（邏輯框架骨架）
  → 命盤圖解 or 九宮格版式
  → 口播 + 畫面字幕同步核心概念
  → 植入行為科學比例（如：「在我曾解過的上百個命盤中，有 78% 的案例⋯」）
  → 決策框架語言替換術語（參見 IP_Design §四 翻譯表）

240–480 秒：案例故事（故事回到行為科學）
  → 感情 / 事業 / 財務 三選一場景
  → 連結觀眾痛點（第五層情緒型關鍵字）
  → 故事→邏輯→再回到情感共鳴的三明治結構

結尾 60 秒：溫暖收場 ＋ CTA
  → 先給溫暖與行動力量（禁止直接推銷跳入）
  → 引導到對應 Playlist（viewer journey）
  → 提示下一支影片＋IG 連結
  ⚑ 收場腳本範例參見 LINARA_IP_Design.md §5.6
```

### Video Chapters 規則（強制，2026 SEO 必備）
> YouTube 官方確認 Chapters 可供創作者自訂，並提供更多內容上下文給演算法；Chapters 標題亦會出現在搜尋結果與 Google Key Moments。

**格式規則（三條強制）：**
1. 描述欄第一個時間戳必須為 `00:00`，後續至少 2 個以上章節
2. 每個 Chapter 最少 10 秒
3. Chapter 標題格式：**用「人會問的問題」或「核心概念詞」**，勿用「段落一、段落二」等流水號

**範例：**
```
00:00 化忌是什麼？努力卻沒有回報的真實原因
01:30 命盤裡的能量消耗點：三種常見落宮位
04:00 案例：職場天花板背後的行為模式
07:30 如何用奇門時間窗突破化忌卡點
```

### 長度標準
| 類型 | 長度 | 適用場景 |
|---|---|---|
| 主星坐命速解 | 6–8 分鐘 | 入口層，搜尋流量 |
| 四化/大限深挖 | 8–12 分鐘 | 深挖層，留存型 |
| 奇門應用案例 | 6–10 分鐘 | 應用層，決策場景 |

### 縮圖規範
- **Linara 虛擬人物半身**（真人比例、風格化分身）＋ **命盤圖或九宮格符號**（二選一作主視覺）
- 文字：≤ 5 個字，單一痛點詞（例：精神內耗？ / 化忌踩雷？）
- 品牌色彩系統（統一標準，不可更改）：
  - 底色：深藍 `#0D0D2B` / 深紫 `#1A1A3A`
  - 文字／邊框：玫瑰金 `#B4918F`
- 禁止：文字堆疊、多色雜亂、無人出鏡的純文字縮圖

### 節奏規則
- 每 30–60 秒切換一次畫面視角（命盤→講解者→關鍵字特寫）
- 字幕：100% 必有，繁體中文書面語為主，英文為輔助（上中下英）

## 五、Playlist 搜尋型流量策略

> 2026 YouTube「Playlist Search」更新讓播放清單本身成為可獨立排名的內容資產。

### 必建四大 Playlist

| # | Playlist 名稱 | 對應關鍵字層 | 目標 |
|---|---|---|---|
| P1 | 紫微主星坐命全集｜Luna's Astral Code | 第一層 | SEO 入口，新觀眾引導 |
| P2 | 四化盲點與化忌系列｜Luna's命盤分析 | 第二層 | 搜尋型穩定流量 |

> **Playlist 最低門檻**：每個 Playlist 至少需要 **3 支影片**，才能被 Playlist Search 有效索引；每個系列累積 **5 支以上**才形成 Topical Authority cluster。

### Playlist 描述模板（每個清單必填）

```
【清單描述模板】

適合：[目標觀眾描述]

🔗 命理諮詢預約：https://www.instagram.com/lunasastralcode
```

### Playlist 流量規則
- 每支新片發佈後 **24 小時內**，必須加入至少一個 Playlist
- 月度排程：每月選 3–5 支舊片，重做縮圖＋標題＋加入新 Playlist＋**更新 End Screen 連結到最新熱門片**（老片復活四步驟）
